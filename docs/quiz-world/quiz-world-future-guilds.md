# 通知型早押しクイズワールド 将来ギルド構想

## 方針

MVPにはギルドを入れない。
最初は1ワールド制で、全員が同じ空間にいる状態を優先する。

ギルドは Season 2 以降の解放機能として検討する。
小集団、チーム、対抗戦、ギルド内ランキングなどを扱うための将来機能である。

表向きの名称は「ギルド」とする。
内部データ名は guilds / guild_members / guild_invites でよい。

## MVPでギルドを入れない理由

- 初期10人では分割するとクイズ体験の密度が落ちる。
- 通知対象抽選が複雑になる。
- 通報、ブロック、招待、退出などの安全設計が増える。
- ギルド対抗戦を入れると、MVPの検証軸が増えすぎる。
- まず1ワールドで通知型早押しが面白いか検証する必要がある。

## ギルド解放段階

| シーズン | ギルド状態 | 説明 |
| --- | --- | --- |
| Season 0 | ギルドなし | 10人の1ワールドで検証。 |
| Season 1 | 運営作成ギルドのみ | 必要に応じて運営が小集団を作る。 |
| Season 2 | 上位ユーザーがギルド作成可能 | 条件を満たしたユーザーに作成権を付与。 |
| Season 3 | ギルド対抗イベント解放 | ギルドランキングや対抗戦を検証。 |

## ギルド機能候補

| 機能 | 内容 |
| --- | --- |
| ギルド内ランキング | ギルドメンバー内で出題・回答ランキングを表示。 |
| ギルド専用クイズ | ギルドメンバーだけに届くクイズ。 |
| ギルド招待 | 招待コードや承認制で参加。 |
| ギルド対抗戦 | ギルド単位で正解数やポイントを競う。 |
| ギルド称号 | シーズン成績や活動に応じた称号。 |
| ギルド貢献度 | 出題、回答、評価などの貢献を可視化。 |
| ギルド作成権 | ランクや安全性条件を満たすと作成できる。 |

## ギルド作成条件候補

ギルド作成は誰でも可能にせず、一定の信頼条件を満たしたユーザーに限定する。

| 条件 | 例 |
| --- | --- |
| 出題者ランク | Lv.3以上。 |
| 回答者ランク | Lv.3以上。 |
| 通報履歴 | 妥当な通報が少ない。 |
| 参加日数 | 30日以上。 |
| 良問評価 | 一定以上。 |
| アカウント状態 | 停止履歴がない、または直近問題がない。 |

## ギルド内ロール案

| ロール | 権限 |
| --- | --- |
| owner | ギルド設定、招待、管理者追加、解散。 |
| admin | 招待、メンバー管理、通報一次対応。 |
| member | 参加、回答、出題。 |

ロールはMVP後の設計であり、初期テーブルには必須ではない。

## ギルドと通知

ギルド導入後は、通知対象の範囲を分ける必要がある。

| 配信範囲 | 説明 |
| --- | --- |
| ワールド配信 | ワールド全体から対象者を抽選。 |
| ギルド配信 | ギルドメンバーから対象者を抽選。 |
| イベント配信 | ギルド対抗戦などのイベント参加者から抽選。 |

将来 quiz_launches に nullable な guild_id を追加することで、ギルド配信を表現できる。

## ギルド安全対策

ギルド機能では、次の安全対策が必要になる。

- ギルド招待拒否
- ギルド退出
- ギルド通報
- ギルド管理者権限
- ギルド管理者の権限濫用対策
- ギルド乱立防止
- ギルド名・説明文の禁止ワードチェック
- ギルド内クイズの通報
- ブロック関係があるユーザー同士の扱い
- 未成年対応を入れるまでは18+を維持

## ギルド対抗戦の方向性

ギルド対抗戦は、ギルド導入後さらに後段の機能とする。

候補ルールは次の通り。

- 一定期間の正解数で競う。
- 難問正解にボーナスを付ける。
- 出題評価もギルドポイントに含める。
- 通報や低評価が多い問題はポイント対象外にする。
- 大人数ギルドが有利になりすぎないよう平均点や上位人数制限を使う。

## データモデル拡張方針

MVPでは worlds / world_members を使い、ギルドは作らなくてもよい。
将来追加する場合は、次を想定する。

- guilds
- guild_members
- guild_invites
- quiz_launches.guild_id nullable
- guild_rank_events
- guild_reports

ギルド導入前に、ワールド単位の通知、通報、ランク、ブロックが安定していることを条件にする。

## ギルド導入前に確認すべきこと

- 1ワールドで通知型早押しが継続利用されているか。
- 出題者ランクが良問作成を促しているか。
- 回答者ランクが参加意欲につながっているか。
- 通知疲れが起きていないか。
- 通報やブロック運用が回っているか。
- ギルドに分けても十分な人数密度があるか。
- 18歳以上限定の運用を継続するか、年齢拡張を検討するか。
