# Phase 10 Participant Guide

## 1. このテストについて

これはQuiz WorldのPreviewテストである。

一般公開ではない。Production環境でもない。Preview環境で、owner/adminが管理できる範囲に限定して、signupからクイズ作成、回答、結果確認、rating、reportまでの流れを確認する。

参加者に伝える前提:

- 18歳以上の方のみ参加できる
- Preview URLはSNSや公開ページに出さない
- Preview URLやinvite codeを第三者へ転送しない
- テスト中のデータは、確認後に削除される可能性がある
- 不適切な投稿、迷惑行為、他者を傷つける投稿は禁止
- 不具合や分かりにくい点があれば報告してもらう

Phase 10ではProduction deploy、Stripe、Web Push、Realtimeは扱わない。Smart Buzzerにも触らない。

## 2. 参加前に必要なもの

参加者に個別共有するもの:

- Preview URL
- 参加者別invite code
- 不具合報告先であるownerへの個別DM

共有しないもの:

- Supabase key
- Vercel env
- service role key
- admin email実値
- DB password
- bypass secret

Preview URL共有範囲は、owner/admin確認後にまず信頼できる1名へ個別DMで広げる。問題がなければ2名目へ拡張する。10人テスト候補全員への共有はまだ行わない。

## 3. Signup手順

1. 個別に受け取ったPreview URLを開く
2. `/signup` を開く
3. emailとpasswordを入力する
4. 18歳以上確認にチェックする
5. 利用規約に同意する
6. プライバシーポリシーに同意する
7. invite codeを入力する
8. signupを実行する

確認してほしいこと:

- 18歳以上確認、terms同意、privacy同意が分かりやすいか
- invite code入力欄が見つけやすいか
- エラーが出た場合、何を直せばよいか分かるか
- signup後に次に進む画面が分かるか

## 4. Login手順

1. `/login` を開く
2. signup時のemail/passwordを入力する
3. loginする
4. `/home` が開けることを確認する

確認してほしいこと:

- login導線が分かりやすいか
- login後にheaderからログインボタンが消えるか
- headerにアカウント導線が表示されるか
- login後に自分が何をすればよいか分かるか
- `/home` に届いたクイズ一覧が見つけやすいか

## 4.1 Account手順

ログイン後、headerの「アカウント」から `/account` を開く。

確認してほしいこと:

- `/account` でパスワード変更への導線が見つかるか
- `/account` にログアウトボタンがあるか
- headerにはログアウトボタンが出ていないか
- `/account/password` で新しいパスワードと確認用パスワードを入力できるか
- 確認用パスワードが一致しない場合、エラーが分かりやすいか

## 5. クイズ作成手順

1. `/create` を開く
2. 四択クイズの問題文を入力する
3. 選択肢を4つ入力する
4. 正解を1つ選ぶ
5. 難易度を選ぶ
6. カテゴリを選ぶ
7. 必要なら下書き保存、またはactiveとして保存する
8. active questionを出題する

確認してほしいこと:

- 四択の入力欄が分かりやすいか
- 正解選択が分かりやすいか
- カテゴリと難易度が選びやすいか
- activeにして出題する導線が分かりやすいか
- エラー時に修正箇所が分かるか

## 6. 届いたクイズへの回答手順

1. `/home` を開く
2. 届いたクイズ一覧を確認する
3. start_at前は問題本文と選択肢が見えないことを確認する
4. start_at後に `/quiz/[launchId]` を開く
5. 選択肢を1つ選ぶ
6. 回答を送信する
7. 回答済み状態になることを確認する

確認してほしいこと:

- `/home` で届いたクイズを見つけやすいか
- start_at前に、まだ問題が見えない理由が分かるか
- 選択肢が押しやすいか
- 回答送信後に、再回答できないことが分かるか

## 7. Result / Rating / Report

### Result

回答後、または締切後に `/result/[launchId]` で結果を確認する。

確認するもの:

- 自分の正誤
- answer_rank
- correct_rank
- 全回答者一覧
- 未回答者一覧
- 正解表示

### Rating

結果画面から、問題を3段階で評価する。

- 良問
- 普通
- 微妙

理由タグは1つだけ選ぶ。現在は評価の変更はできない。

### Report

不適切、答えが曖昧、その他問題があると感じた場合は、結果画面からreportする。

reportはadminが確認する。迷った場合でも、問題があると思ったらreportしてよい。

## 8. 不適切投稿禁止

以下は禁止する。

- 差別、嫌がらせ、脅迫、性的・暴力的な投稿
- 個人情報の投稿
- 他者を攻撃する問題文や選択肢
- 正解が意図的に曖昧な問題
- テスト目的を超えた大量投稿
- Preview URLやinvite codeの公開

問題がある投稿を見つけた場合は、report機能で通報する。

## 9. 既知制約

Phase 10開始時点の既知制約:

- Web Pushなし
- Realtimeなし
- `/home` のポーリング中心
- profile rank説明は最小表示
- `/world` 補助指標は最小表示
- Preview URLはDeployment Protection配下
- Production環境ではない
- テストデータは削除される可能性がある

## 10. 不具合報告テンプレート

不具合や分かりにくい点は、ownerへの個別DMで以下の形で報告する。

```text
画面名:
操作手順:
期待した結果:
実際の結果:
エラー文:
スクリーンショット:
発生日時:
使った端末/ブラウザ:
補足:
```

報告例:

```text
画面名: /create
操作手順: 問題文と選択肢4つを入力して保存した
期待した結果: 作成済み問題一覧に表示される
実際の結果: エラーが出て保存できなかった
エラー文: choices must be unique
スクリーンショット: あり
発生日時: 2026-05-28 20:30頃
使った端末/ブラウザ: iPhone / Safari
補足: 選択肢に似た文言を入れていた
```

## 11. 参加者への案内文

個別共有時は、以下をベースにする。

```text
Quiz WorldのPreviewテストに協力してほしいです。

これは一般公開前のPreview環境です。Productionではありません。
テスト中のデータは、確認後に削除される可能性があります。

参加条件:
- 18歳以上の方のみ利用できます
- Preview URLとinvite codeをSNSや公開ページに出さないでください
- 不適切な投稿や迷惑行為は禁止です
- 不具合や分かりにくい点があれば報告してください

試してほしいこと:
1. signup / login
2. 四択クイズの作成
3. 届いたクイズへの回答
4. 結果画面の確認
5. rating / report
6. profileのscore / rank確認

不適切な問題を見つけた場合は、結果画面のreportから通報してください。
報告時は、画面名、操作手順、期待した結果、実際の結果、スクリーンショット有無を教えてください。
```

Preview URL、参加者別invite code、不具合報告先は個別共有時に追記する。docsやREADMEには実値を書かない。
