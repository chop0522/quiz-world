# 通知型早押しクイズワールド ランキング・ランク設計

## 目的

ランク設計の目的は、良いクイズを作る行動と、速く正確に答える行動を分けて評価することである。

- 出題者は、良問を継続的に作るほど配信できる人数と出題回数が増える。
- 回答者は、正解率、速さ、難問正解、継続参加で評価される。
- 課金で配信人数を増やす設計にはしない。
- 出題者の配信人数増加と、ワールド全体の参加枠増加は別概念にする。

## 出題者ランク

出題者ランクは、他のプレイヤーに良いクイズ体験を届けているかを評価する。

### 評価要素

| 要素 | 加点・評価方針 |
| --- | --- |
| 良問評価 | 回答者からの高評価が多いほど加点。 |
| 回答参加率 | 通知対象者のうち、実際に回答した割合が高いほど加点。 |
| 正解率の適正さ | 極端に簡単すぎる、難しすぎる問題ばかりは抑制。 |
| 通報率の低さ | 通報率が低いほど信頼度が上がる。 |
| 答えが曖昧評価の少なさ | 「正解が曖昧」「選択肢が不適切」などの低評価を抑制。 |
| 離脱率の低さ | 通知後に離脱されにくい問題を評価。 |
| 継続的な良問出題 | 短期的な大量出題より、継続した良問を評価。 |

### 出題者ランク別制限例

| 出題ランク | 1日の出題回数 | 1回の配信人数 | 想定 |
| --- | ---: | ---: | --- |
| Lv.0 | 1 | 3 | 初期状態。少人数にだけ配信できる。 |
| Lv.1 | 2 | 5 | 良問評価が少し集まった状態。 |
| Lv.2 | 3 | 8 | 安定した出題者。初期10人ワールドではほぼ全体に近い配信が可能。 |
| Lv.3 | 5 | 12 | ワールド拡大後に影響力が増える。 |
| Lv.4 | 8 | 20 | 信頼された上位出題者。 |

MVPが10人制の間は、配信人数の上限はワールドの有効メンバー数と通知対象条件によってさらに制限される。

### 出題者スコア例

| イベント | 点数例 |
| --- | ---: |
| 高評価を受けた | +5 |
| 回答参加率が高かった | +3 |
| 正解率が適正範囲だった | +2 |
| 難問として高評価だった | +3 |
| 通報が妥当と判断された | -10 |
| 曖昧な問題として低評価が多かった | -5 |
| 通知後の離脱率が高かった | -3 |

点数は初期案であり、MVP中に実データを見て調整する。

## 回答者ランク

回答者ランクは、早く正確に答える力と、継続的に参加する姿勢を評価する。

### 評価要素

| 要素 | 加点・評価方針 |
| --- | --- |
| 正解数 | 基本加点。 |
| 正解率 | 一定以上の回答数を超えてから重視。 |
| 正解者順位 | correct_rank が上位ほど加点。 |
| 難問正解ボーナス | 難易度が高く、正解率が低い問題の正解に加点。 |
| 連続参加ボーナス | 継続参加に軽く加点。 |
| 見逃し | 強いペナルティは与えない。 |
| 誤答 | 軽めのペナルティにする。 |

通知型ゲームでは、毎回参加できないことが自然に起こる。そのため、見逃しには強いペナルティを与えない。

### 回答者スコア例

| イベント | 点数例 |
| --- | ---: |
| 回答した | +1 |
| 正解した | +5 |
| correct_rank 1位 | +5 |
| correct_rank 2位 | +3 |
| correct_rank 3位 | +2 |
| 難問に正解 | +3 |
| 連続参加 | +1 |
| 誤答 | -1 |
| 回答対象だったが未回答 | 0 |

早い不正解は answer_rank では上位でも、回答者ランクには大きく加点しない。

## 順位定義

| 項目 | 定義 | ランクへの影響 |
| --- | --- | --- |
| answer_rank | 全回答者のサーバー受信順 | 参考情報として表示。回答速度の演出に使う。 |
| correct_rank | 正解者だけのサーバー受信順 | 回答者ランクに強く反映。 |

順位判定では端末時刻を使わない。サーバーが受信した answer_received_at を唯一の基準にする。

## ワールド成長

ワールド全体の成長は、個人の出題ランクとは別に扱う。

| シーズン | 参加枠 | 状態 |
| --- | ---: | --- |
| Season 0 | 10人 | 初期検証。小さなクイズ村。 |
| Season 1 | 15人 | 通知・回答・評価が安定した段階。 |
| Season 2 | 20人 | ギルド準備を始められる段階。 |
| Season 3 | 30人 | ギルドやイベントの検証候補。 |
| Season 4 | 50人 | 本格的なコミュニティ設計の前段階。 |

## 参加枠解放条件例

参加枠は、ワールド全体が健全に活動している場合に解放する。

| 条件 | 例 |
| --- | --- |
| 累計出題数 | 100問以上。 |
| 累計回答数 | 500回答以上。 |
| 平均クイズ評価 | 3.8/5以上など。 |
| 通報率 | 一定以下。 |
| 継続日数 | 14日以上継続稼働。 |
| 上位出題者数 | Lv.2以上が3人以上。 |
| 上位回答者数 | Lv.2以上が3人以上。 |

MVPでは自動解放ではなく、管理者判断でシーズンを進める運用でもよい。

## 重要な分離

出題者の配信人数増加と、ワールド全体の参加枠増加は別概念である。

| 概念 | 増えるもの | 増える条件 |
| --- | --- | --- |
| 出題ランク | 1日の出題回数、1回の配信人数 | 良問評価、回答参加率、低通報率など。 |
| ワールド成長 | サービス全体の参加枠 | 累計活動、安全性、継続性、上位ユーザー数など。 |

課金で配信人数を増やす設計にはしない。配信人数は評価によって増える。
参加枠はワールド全体の成長によって増える。

## ランク運用上の注意

- 初期はスコア計算を単純にし、運用しながら調整する。
- 通報や低評価による減点は、誤用を避けるため一定数以上のシグナルを見て反映する。
- 評価者の偏りを避けるため、同じユーザーからの評価を過大に効かせない。
- 出題者ランクは通知体験に直接影響するため、上げる条件より下げる条件を慎重に設計する。
- 回答者ランクは見逃しに寛容にし、参加できるときに参加したくなる設計にする。
