# 通知型早押しクイズワールド 安全設計・年齢ポリシー

## 基本方針

MVPは18歳以上限定の少人数招待制サービスとして開始する。
18歳未満は保護者同意があってもMVPでは利用不可とする。

理由は、UGC、通知、ランキング、評価、通報、将来的な課金可能性があるためである。
MVPでは未成年者対応の運用負荷を避け、クイズ体験と安全運用の検証に集中する。

## 年齢制限

| 項目 | 方針 |
| --- | --- |
| MVP対象年齢 | 18歳以上限定。 |
| 18歳未満 | 利用不可。 |
| 保護者同意 | MVPでは受け付けない。 |
| 生年月日保存 | MVPでは保存しない。 |
| 年齢確認方法 | 登録時に「18歳以上である」チェックを必須にする。 |
| 保存する情報 | age_confirmed_at を保存する。 |

登録時には、18歳以上確認に加えて、利用規約とプライバシーポリシー同意を必須にする。

profiles には次を持つ。

- age_confirmed_at
- terms_accepted_at
- privacy_accepted_at

## 18歳以上限定にする理由

- ユーザーがクイズを作るUGCサービスである。
- 通知が中心体験であり、時間帯への配慮が必要である。
- ランキングや評価がユーザー間の比較を生む。
- 通報やブロックなどの安全運用が必要である。
- 将来的に課金を検討する余地がある。
- 未成年者取消しや保護者同意フローを避ける必要がある。
- 少人数MVPでは、未成年対応まで含めると運用負荷が高すぎる。

## 将来16〜17歳を許可する場合の追加設計

MVPでは対応しない。
将来16〜17歳を許可する場合は、少なくとも次の設計が必要になる。

| 領域 | 必要な追加設計 |
| --- | --- |
| 課金 | 課金不可、または保護者同意フロー。 |
| 通知 | 深夜通知制限、通知時間制限。 |
| プロフィール | 公開範囲制限、検索制限。 |
| 問い合わせ | 保護者問い合わせ対応。 |
| 法務 | 年齢別プライバシーポリシー。 |
| コミュニケーション | DMなし、自由チャットなしを維持。 |
| 管理 | 年齢区分に応じた通報対応。 |

15歳以下は一般版では対象外、または保護者・学校管理版のみ検討とする。

## UGCリスク

クイズはユーザー生成コンテンツであるため、次のリスクがある。

- 不適切な問題文
- 差別的・攻撃的な表現
- 成人向け・暴力的な表現
- 個人情報の投稿
- 正解が曖昧な問題
- 誤情報
- 著作権上問題のある引用
- 特定ユーザーへの嫌がらせ目的の出題
- 通知を悪用した迷惑行為

MVPでは自由チャットやコメントを入れず、UGC対象をクイズに限定して安全範囲を狭める。

## UGC対策

| 対策 | 説明 |
| --- | --- |
| 通報 | 問題、配信、出題者を通報できる。 |
| ブロック | ブロック関係では通知対象から除外する。 |
| ミュート | 将来的に出題者単位の通知抑制として検討。 |
| 不適切クイズの配信停止 | 管理者が問題の配信を止められる。 |
| アカウント停止 | 悪質なユーザーを停止できる。 |
| 出題回数制限 | 低ランク出題者の配信影響を小さくする。 |
| 低評価問題の配信制限 | 評価が悪い問題を再配信しない。 |
| 通報率が高い出題者の制限 | 配信人数や出題回数を制限する。 |
| 禁止ワード | 明確に不適切な語を検知する。 |
| 管理者レビュー | 通報や低評価が多い問題を確認する。 |
| 通報理由の分類 | 対応優先度を決めやすくする。 |

## MVP安全機能

MVPで必須にする安全機能は次の通り。

- 通報
- ブロック
- 通知OFF
- quiet hours
- 1日通知上限
- 深夜通知デフォルトOFF
- 休憩モード
- 出題回数制限
- 低評価問題の配信制限
- 管理者によるクイズ停止
- 管理者によるユーザー停止

## MVPに入れない安全リスクの高い機能

| 機能 | 入れない理由 |
| --- | --- |
| DM | 個別トラブルや未成年対応リスクが高い。 |
| 自由チャット | モデレーション範囲が広がる。 |
| 公開コメント | クイズ以外のUGCが増える。 |
| 不特定多数のタイムライン | SNS化し、安全運用が重くなる。 |
| 課金による通知数増加 | 迷惑通知やPay to Winに見える可能性がある。 |

## 通報理由の分類案

| 分類 | 例 |
| --- | --- |
| 不適切表現 | 差別、攻撃、性的表現、暴力表現。 |
| 個人情報 | 氏名、住所、電話番号、SNS IDなど。 |
| 正解が曖昧 | 正解が複数ある、選択肢が不十分。 |
| 誤情報 | 明らかに誤った内容。 |
| 著作権 | 長文引用、無断転載の疑い。 |
| 嫌がらせ | 特定個人や集団を狙った内容。 |
| その他 | 上記に当てはまらない問題。 |

## 通知安全ポリシー

通知はゲーム体験の中心だが、ユーザーの生活を妨げてはいけない。

- quiet hours をデフォルトで設定する。
- 深夜通知はデフォルトOFFにする。
- 1日の通知上限を設ける。
- 休憩モードを用意する。
- 通知失敗や未開封を強く罰しない。
- 通知を増やす手段を課金にしない。

## 管理者対応方針

管理者は次を行えるようにする。

- 通報一覧の確認
- 問題の配信停止
- 問題のステータス変更
- ユーザーの一時停止
- 招待コードの停止
- ウェイトリスト承認
- 参加枠の変更

MVP初期では管理画面を簡易にしてもよいが、運用権限と監査ログの考え方は最初から分けておく。
